# Feature Specification: 개인정보 처리방침 웹 페이지

**Feature Branch**: `005-privacy-policy-web`
**Created**: 2026-02-15
**Status**: Draft
**Input**: User description: "앱스토어 심사용 개인정보 처리방침 웹 페이지를 Expo Web으로 구현"

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 앱스토어 심사관의 개인정보 처리방침 확인 (Priority: P1)

앱스토어 심사관이 앱 제출 시 제공된 개인정보 처리방침 URL에 접속하여 앱의 개인정보 수집 및 처리 내용을 확인한다.

**Why this priority**: 앱스토어 심사 통과를 위한 필수 요건이며, 이 페이지가 없으면 앱 제출 자체가 불가능하다.

**Independent Test**: 개인정보 처리방침 URL에 웹 브라우저로 접속하여 페이지가 정상 표시되는지 확인하면 된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 웹 브라우저를 열고, **When** 개인정보 처리방침 URL에 접속하면, **Then** 개인정보 처리방침 전문이 한국어로 표시된다.
2. **Given** 개인정보 처리방침 페이지가 로드되었을 때, **When** 페이지 내용을 확인하면, **Then** 앱 이름(Cursed By Jesus), 개발자 정보, 수집하는 개인정보 항목, 연락처가 포함되어 있다.
3. **Given** 개인정보 처리방침 페이지가 로드되었을 때, **When** 페이지 하단을 확인하면, **Then** 시행일자가 명시되어 있다.

---

### User Story 2 - 앱 사용자의 개인정보 처리방침 열람 (Priority: P2)

앱 사용자가 앱 내 설정 또는 외부 링크를 통해 개인정보 처리방침을 열람하고 자신의 개인정보가 어떻게 처리되는지 확인한다.

**Why this priority**: 사용자 신뢰 확보와 법적 요건 준수를 위해 필요하지만, 앱스토어 심사 통과보다 우선순위가 낮다.

**Independent Test**: 모바일 브라우저에서 URL에 접속하여 가독성과 반응형 레이아웃이 정상 동작하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 모바일 기기에서, **When** 개인정보 처리방침 URL에 접속하면, **Then** 모바일 화면에 맞게 가독성 있는 레이아웃으로 표시된다.
2. **Given** 사용자가 페이지를 읽고 있을 때, **When** 특정 섹션을 찾으려 하면, **Then** 명확한 제목과 구조화된 섹션으로 원하는 정보를 쉽게 찾을 수 있다.

---

### Edge Cases

- 인터넷 연결이 느린 환경에서 페이지가 접속될 경우, 텍스트 기반 콘텐츠이므로 빠르게 로드되어야 한다.
- 다양한 브라우저(Safari, Chrome, Firefox)에서 접속 시 동일하게 표시되어야 한다.
- 스크린 리더 등 접근성 도구를 사용하는 사용자도 내용을 읽을 수 있어야 한다.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 개인정보 처리방침 페이지는 고유한 공개 URL을 통해 접근 가능해야 한다.
- **FR-002**: 페이지에는 앱 이름("Cursed By Jesus"), 서비스 제공자 정보가 표시되어야 한다.
- **FR-003**: 페이지에는 수집하는 개인정보 항목을 명시해야 한다 (현재 앱은 개인정보를 수집하지 않으므로 "수집하지 않음"을 명확히 기재).
- **FR-004**: 페이지에는 앱의 데이터 저장 방식(기기 내 로컬 저장, 서버 전송 없음)을 설명해야 한다.
- **FR-005**: 페이지에는 개인정보 처리방침 문의를 위한 연락처(이메일)를 포함해야 한다.
- **FR-006**: 페이지에는 개인정보 처리방침의 시행일자를 명시해야 한다.
- **FR-007**: 페이지는 데스크톱과 모바일 환경 모두에서 가독성 있게 표시되어야 한다 (반응형).
- **FR-008**: 페이지는 정적 콘텐츠로 구성되며, 별도의 사용자 입력이나 인증 없이 누구나 접근 가능해야 한다.
- **FR-009**: 페이지 콘텐츠는 한국어로 작성되어야 한다.

### Assumptions

- 앱 "Cursed By Jesus"는 현재 개인정보를 수집하지 않으며, 모든 데이터(성인 정보, 즐겨찾기)는 기기 내 SQLite에 로컬 저장된다.
- 서버로 사용자 데이터를 전송하지 않는다.
- 앱스토어 심사 시 개인정보 처리방침 URL이 활성 상태여야 한다.
- 연락처 이메일은 개발자가 별도 지정한다.
- 한국어 단일 언어로 작성하며, 영어 버전은 향후 필요 시 추가한다.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 개인정보 처리방침 URL에 접속 시 3초 이내에 전체 콘텐츠가 표시된다.
- **SC-002**: 앱스토어 심사 제출 시 개인정보 처리방침 URL이 유효하고 정상 접근 가능하여 심사를 통과한다.
- **SC-003**: 모바일(320px~428px)과 데스크톱(1024px 이상) 해상도에서 텍스트가 잘리거나 겹치지 않고 가독성을 유지한다.
- **SC-004**: 페이지가 한국 개인정보보호법(PIPA)의 필수 고지 항목을 포함한다.
