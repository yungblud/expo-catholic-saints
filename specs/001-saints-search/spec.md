# Feature Specification: Catholic Saints Search

**Feature Branch**: `001-saints-search`
**Created**: 2026-01-28
**Status**: Draft
**Input**: User description: "카톨릭 성인들을 검색하는 앱을 만들꺼야"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Search Saints by Name (Priority: P1)

사용자가 성인의 이름을 입력하여 해당 성인의 정보를 검색하고 확인할 수 있다.

**Why this priority**: 이름 검색은 성인 검색 앱의 가장 기본적이고 핵심적인 기능이다. 사용자가 특정 성인에 대해 알고 싶을 때 가장 먼저 시도하는 방법이다.

**Independent Test**: 검색창에 "프란치스코"를 입력하고 아시시의 성 프란치스코 정보가 표시되는지 확인하여 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 앱이 실행된 상태에서, **When** 사용자가 검색창에 "베네딕토"를 입력하면, **Then** 이름에 "베네딕토"가 포함된 모든 성인 목록이 표시된다
2. **Given** 검색 결과 목록이 표시된 상태에서, **When** 사용자가 특정 성인을 선택하면, **Then** 해당 성인의 상세 정보(축일, 수호성인 분야, 약력)가 표시된다
3. **Given** 검색창에 텍스트가 입력된 상태에서, **When** 사용자가 검색어를 수정하면, **Then** 결과가 실시간으로 업데이트된다

---

### User Story 2 - Browse Saints by Feast Day (Priority: P2)

사용자가 특정 날짜(오늘 또는 선택한 날짜)의 축일 성인을 확인할 수 있다.

**Why this priority**: 많은 카톨릭 신자들이 매일 그 날의 성인을 묵상하고 기도한다. 날짜별 검색은 일상적인 신앙생활에 직접적으로 도움이 된다.

**Independent Test**: 앱을 열고 오늘 날짜의 축일 성인 목록이 자동으로 표시되는지 확인하여 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 앱이 실행되면, **When** 메인 화면이 로드되면, **Then** 오늘 날짜의 축일 성인들이 표시된다
2. **Given** 날짜 선택 기능이 있을 때, **When** 사용자가 12월 25일을 선택하면, **Then** 12월 25일이 축일인 성인들의 목록이 표시된다
3. **Given** 축일 성인 목록이 표시된 상태에서, **When** 사용자가 성인을 선택하면, **Then** 해당 성인의 상세 정보가 표시된다

---

### User Story 3 - Search by Patronage (Priority: P3)

사용자가 특정 분야(직업, 상황, 질병 등)의 수호성인을 검색할 수 있다.

**Why this priority**: 수호성인 검색은 특정 상황에서 기도하고자 할 때 유용하다. 핵심 기능은 아니지만 앱의 가치를 크게 높인다.

**Independent Test**: "학생" 또는 "여행자"를 검색하여 해당 분야의 수호성인이 표시되는지 확인하여 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 수호성인 검색 모드에서, **When** 사용자가 "의사"를 입력하면, **Then** 의사/의료인의 수호성인 목록이 표시된다
2. **Given** 수호성인 카테고리 목록이 있을 때, **When** 사용자가 "직업" 카테고리를 선택하면, **Then** 직업별 수호성인 목록이 표시된다

---

### Edge Cases

- 검색어와 일치하는 성인이 없을 때 어떻게 처리하는가? → "검색 결과가 없습니다" 메시지와 함께 유사한 검색어 제안
- 네트워크 연결이 없을 때 어떻게 처리하는가? → 오프라인 모드에서 캐시된 데이터 제공 또는 안내 메시지 표시
- 동명이인 성인이 여러 명 있을 때 어떻게 구분하는가? → 축일과 별칭(예: "아시시의")을 함께 표시하여 구분
- 한글/영문 혼용 검색 시 어떻게 처리하는가? → 한글 이름과 라틴어/영어 이름 모두 검색 가능

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 성인 이름으로 검색할 수 있어야 한다 (한글, 영어, 라틴어 지원)
- **FR-002**: 시스템은 검색어 입력 시 실시간으로 결과를 필터링해야 한다
- **FR-003**: 시스템은 각 성인의 상세 정보를 표시할 수 있어야 한다 (이름, 축일, 수호 분야, 약력)
- **FR-004**: 시스템은 날짜별로 축일 성인을 조회할 수 있어야 한다
- **FR-005**: 시스템은 수호 분야별로 성인을 검색할 수 있어야 한다
- **FR-006**: 시스템은 검색 결과가 없을 때 적절한 피드백을 제공해야 한다
- **FR-007**: 시스템은 자주 검색하는 성인을 즐겨찾기에 저장할 수 있어야 한다
- **FR-008**: 시스템은 오프라인에서도 기본적인 검색 기능을 제공해야 한다

### Key Entities

- **Saint (성인)**: 카톨릭 성인 정보를 나타냄. 주요 속성: 이름(한글/영어/라틴어), 축일, 수호 분야, 약력, 출생/사망 연도, 시성 연도
- **Feast Day (축일)**: 특정 날짜에 기념하는 성인의 날. 월/일 정보와 해당 성인 목록과 연결됨
- **Patronage (수호 분야)**: 성인이 수호하는 분야. 카테고리(직업, 지역, 상황, 질병 등)와 구체적인 항목으로 구성
- **Favorite (즐겨찾기)**: 사용자가 저장한 성인 목록. 사용자 기기에 로컬 저장

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 3초 이내에 원하는 성인을 검색하여 찾을 수 있다
- **SC-002**: 검색 결과가 입력 후 0.5초 이내에 표시된다
- **SC-003**: 앱에서 최소 1,000명 이상의 성인 정보를 제공한다
- **SC-004**: 사용자의 80% 이상이 첫 시도에서 원하는 성인을 찾을 수 있다
- **SC-005**: 오프라인 상태에서도 캐시된 성인 정보에 접근할 수 있다
- **SC-006**: 앱 최초 실행 시 5초 이내에 오늘의 축일 성인이 표시된다

## Assumptions

- 성인 데이터는 공개적으로 사용 가능한 카톨릭 성인 데이터베이스 또는 API를 활용한다
- 한국 카톨릭 사용자를 주요 대상으로 하므로 한글 인터페이스가 기본이다
- 로마 카톨릭 전례력 기준의 축일을 사용한다
- 사용자 계정/로그인 없이 즐겨찾기는 기기 로컬에 저장한다
